# Docker Registries

A **Docker Registry** is a storage and distribution service for Docker images.

It acts as an online library where Docker images are stored, managed, and shared.

---

## Types of Registries

### Public Registry

Public registries are open to everyone.

* Anyone can download images.
* Developers can share images with the community.
* Ideal for open-source projects.

**Example:** Docker Hub

### Private Registry

Private registries are secure and access-controlled.

* Only authorised users can access images.
* Suitable for organisations and private projects.
* Can be hosted in the cloud or on-premises.

---

## Benefits of Docker Registries

* **Streamlined Deployment** – Once an image is stored in a registry, it can be easily accessed and deployed from anywhere.
* **Improved Collaboration** – Teams can share and manage images through a centralised registry.
* **Consistency Across Environments** – Every developer, test environment, and production server uses the same image, reducing configuration issues.
* **Version Control** – Multiple versions (tags) of an image can be stored, making it easy to roll back to previous releases.
* **Centralised Storage** – Images are stored in one location, making them easier to organise and maintain.

---

## Popular Docker Registries

* **Docker Hub** – The default public Docker registry.
* **GitHub Container Registry (GHCR)** – Stores container images alongside GitHub repositories.
* **Amazon Elastic Container Registry (ECR)** – Managed registry for AWS.
* **Google Artifact Registry** – Container registry for Google Cloud.
* **Azure Container Registry (ACR)** – Microsoft's private container registry service.
